¿How Does Authentication Work?

En la construccion de API's no de trabaja con sesiones, porque RESTFULL api's son 'stateless', no nos
preocupamos por el cliente, ya que se intenta desacoplar al cliente-servidor de la manera mas estricta
posible. Teniendo en cuenta como criterio de diseño que cada peticion debe ser manejada de forma indepe-
ndiente. Eso significa que cada peticion debe tener la informacion suficiente para autenticarse en 
el servidor.

Cliente
|
|(Client sends auth data)
|(Server sends a TOKEN generated by the server)
|
Servidor

El cliente guarda el TOKEN con cualquier mecanismo de Storage
El servidor es quien valida el TOKEN en cada peticion, es indescifrable ya que utiliza un algoritmo unico

¿Que es TOKEN?
JSON WEB TOKEN (JWT)= JSON data + Signature(generada y validada por el server)
